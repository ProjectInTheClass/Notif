어떻게 만들었나요?

Amazon Web Service 를 통한 게시글 수집

  • Lambda
  • CloudWatch
  • API Gateway
  • RDS

AWS의 CloudWatch와 Lambda 를 이용해 특정 시간마다 저희가 서비스하고 있는 사이트들을 전부 크롤링하여 데이터를 수집합니다. RDS와 API Gateway를 통해 DB에 있는 정보의 접근을 용이하게 했습니다. 앱을 열게 되면, 디바이스에서 서버에 최근 추가된 게시글들을 요청합니다. 서버를 통해 데이터를 받으면 사용자가 설정한 채널과 태그에 따라 게시글을 분류하여 볼 수 있도록 해줍니다.

왜 Amazon Web Service 인가요?

저렴한 비용

AWS는 사전 확약금이나 장기 약정 없이 저렴한 종량 과금제 방식으로 운영됩니다. 큰 서비스가 아닌 경우는 무료로 이용할 수 있을정도로 제작 비용을 아낄 수 있습니다.

접근성

AWS는 언어 및 운영 체제에 구애받지 않는 플랫폼입니다. 상황에 따라 다양한 개발 플랫폼 또는 프로그래밍 모델을 선택할 수 있습니다.

개발 속도 향상

서버를 구축하는 시간이 사라져, 앱 개발에 좀 더 집중할 수 있습니다.
